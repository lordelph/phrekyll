Deploy Using Git's post-update hook

Git supports various hooks, which are basically CLI scripts that are executed at different points of git workflow.

For easy site deployment we can use post-update hook, which is executed on our remote production host everytime we push branches in our local/dev copy of Phrekyll site.

The algorithm is simple:

  1. Create a Git repository of your Phrekyll project.
  2. Clone that repository on production host.
  3. Activate post-update hook on production host, it will be responsible for pulling/updating the site everytime something is pushed to project repository.
  4. Clone repository you created at step 1 into your dev box, where you can update site as you see suitable.

Now when you update the code on your dev box, then git-commit and git-push it to production host, once push is done post-update hook is invoked, which basically contains instructions to syncronize by pulling changes.

You should note that automatic pushing to live production server is something that is frowned upon unless you know what you are doing. Basically, you must understand that once you push changes they are live on your production host. If that's what you want - got with it!
