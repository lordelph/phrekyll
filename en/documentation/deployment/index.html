Project Deployment

 

Since Phrekyll generates simple HTML pages as output, pushing project to your web host is trivial:
All you need to do is to copy over generated files.

Depending on your needs, there are several ways to automate this process.

Manual Deployment

Supposing that Phrekyll directory is controlled by Git and you have shell access to production server, most straight-forward approach will be:

  1. Compile site with phr up.
  2. git-commit and git-push to some central repository (say GitHub).
  3. On production server, git-pull the update from that central repository.

Obviously, the same approach can be used with Subversion or any other SCM tool. This process can be further automated using some build tool, such as Phing.

Alternatively, you can phr up directly on production server (if you have full control over it, so that Phrekyll can be installed there).

Synchronize using rsync

Again, since Phrekyll site is just a collection of static pages, you can use rsync. Assuming that generated site lives in public_html on your developer box, here is simple shell script to push files to remote production server:

#!/usr/bin/env bash
phr up && rsync -avz --delete public_html/ my:/path/on/remote/host
